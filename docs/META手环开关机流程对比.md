## 0. TL;DR

> 数据均以 **Meta 官方腕带说明书**（用户提供原文）为准；早先基于网页摘要的推断已用 ⚠ 标注否决。

| 维度 | Meta Neural Band | 本项目 (nRF54L15) | 关键差异 |
|---|---|---|---|
| 开机 / 唤醒长按 | **1s**（白灯亮起并脉动） | 3s | Meta 短按即开，靠"必须先充电"防误触 |
| 配对模式长按 | **3s**（白灯闪烁；配对成功后转常亮） | 不存在（开机即广告） | Meta 配对态独立可视化 |
| 关机长按 | **6s**（灯光逐渐熄灭） | 8s（短振 2 次） | 长度接近，Meta 用渐暗 LED 反馈 |
| 恢复出厂长按 | **16s**（橙 + 白脉动） | 15s（白黄交替 + 短振 3 次） | 长度接近 |
| 短按 | **查电池**（绿常亮 / 橙常亮 / 橙闪 = 满 / 未满 / 低电） | STANDBY → WAITING（无电量显示） | 完全不同语义 |
| 自动关机判据 | **IMU 无运动 1h** | 纯时间 3 min + 5 min | Meta 用 motion 判据，避免静止误关 |
| LED 颜色 | 白 / 橙 / 绿 / 橙白脉动（出厂） | RGB 全色 + 软件 PWM | Meta 三色就够用 |
| 上行数据 | 仅事件（本地推理） | 原始 EMG + IMU 全量 | 阶段不同（部署态 vs 算法开发期） |

## 2. Meta Neural Band 开关机行为（以官方说明书为准）

### 2.1 按键 → 动作 映射表（官方权威）

| 操作 | 时长 | 视觉反馈 | 说明 |
|---|---|---|---|
| 短按（按下并松开） | < 1s | 显示电量 LED | 不切换状态，仅查询 |
| **开机 / 唤醒** | **1s** | 白灯亮起并开始脉动 | 从 OFF / AUTO-OFF 都用此操作 |
| **配对模式** | **3s** | 白灯闪烁 → 配对成功后白光常亮 | 显式触发可被发现窗口 |
| **关机** | **6s** | 灯光逐渐熄灭 | 显式关机，有渐暗反馈 |
| **恢复出厂** | **16s** | 橙光 + 白光脉动 | 故障兜底 |
| 自动关机 | — | — | 1h 无 IMU 运动（来自 Meta Help 使用说明） |

### 2.2 LED 电量指示（短按触发）

| LED | 含义 |
|---|---|
| 常亮绿灯 | 已充满 |
| 常亮橙灯 | 未充满 |
| 闪烁橙灯 | 电量低 |

> 颜色至少包含 **白 / 绿 / 橙** 三色 + 出厂的 **橙白脉动** 组合；**不是先前推断的"单点白光"**。

### 2.3 修正说明（⚠ 已否决的早先推断）

| 早先说法 | 实际 | 来源 |
|---|---|---|
| 首次开机 2s | **1s** | 官方说明书 |
| 唤醒 1s 与开机 2s 是两套时长 | **统一 1s**（开机 = 唤醒） | 官方说明书 |
| 无显式关机 | **有：长按 6s** | 官方说明书 |
| 单点白光 LED | **白 + 绿 + 橙 + 橙白脉动** | 官方说明书 |
| 关机无用户提示 | **灯光逐渐熄灭** | 官方说明书 |

### 2.4 用户旅程

```
首次开箱（必须先充 ≥30s）
    │
    │ 长按 1s
    ▼
开机（白光脉动） ── 长按 3s ──▶ 配对模式（白光闪烁）
                                       │
                                       │ Meta AI app 配对成功
                                       ▼
                                  CONNECTED（白光常亮）
                                       │
         ┌─────────────────────────────┴────────────────────┐
         │                                                  │
  用户长按 6s                                       1h 无 IMU 运动
         │                                                  │
         ▼                                                  ▼
      关机（灯渐暗）                                 AUTO-OFF（静默）
         │                                                  │
         └────────────── 下次长按 1s 唤醒 ──────────────────┘
```

### 2.5 设计哲学

1. **单按钮 + 多色 LED + 振动**：硬件交互极简但 LED 信息密度足够（4 种颜色态）
2. **统一长按时长阶梯（1 / 3 / 6 / 16 s）**：递进式语义，越破坏性越长
3. **配对态独立可视化**：白闪 vs 白常亮 = 配对中 vs 已连接
4. **运动 + 显式关机双判据**：用户主动 6s 可关；不操作也会 1h 自动关
5. **充电与开机解耦**：插 USB-C 不唤醒主控
6. **本地推理 + 事件上行**：BLE 带宽极低，~ 18h 续航

## 3. 本项目当前流程速览

> 详细见 `docs/开关机与状态机逻辑.md`，本节只摘要对比要点。

### 3.1 状态与触发

| 状态 | 触发 / 超时 | LED | 说明 |
|---|---|---|---|
| **OFF** | 8s 长按关机 / 5min STANDBY 超时 / 出厂 | 全灭 | PMIC ship mode，~ 0.5 µA |
| **STARTING** | 3s 长按开机（POR + boot magic + SHPHLD 事件双校验） | 黄常亮 + 振动 1 次 | 2s 自动跳 WAITING |
| **WAITING_CONNECT** | STARTING 完成 / WORKING 断连 / STANDBY 短按 | 白心跳（1s 亮 / 1s 灭） | BLE 广告中 |
| **WORKING** | BLE 已连接 | 全灭（省电） | EMG + IMU 全量上行 |
| **STANDBY** | WAITING 3min 无连接 | 黄心跳（1s 亮 / 1s 灭） | 停采集，5min 后 OFF |

### 3.2 长按层级

| 时长 | 动作 |
|---|---|
| < 50 ms | 消抖丢弃 |
| 50 ms ~ 3s 后松手 | 短按：STANDBY → WAITING |
| ≥ 3s（OFF） | 开机 |
| ≥ 8s | 立即关机（短振 2 次） |
| ≥ 15s | 恢复出厂（白黄交替 + 短振 3 次 + 冷重启） |

### 3.3 数据流定位

当前阶段 = **算法开发期**：设备做高质量 8 通道 EMG + 6 轴 IMU 数据源，原始信号 BLE 上行到上位机做 AI 训练（非 Meta 那种"仅事件"模式）。

## 4. 并排对比（核心维度）

### 4.1 开机与唤醒

| 项 | Meta | 本项目 | 评注 |
|---|---|---|---|
| 首次开机 / 再开机 | 长按 **1s** | 长按 3s | Meta 靠"需充电 ≥30s"作为误触一道闸 |
| 自动关机后唤醒 | 长按 **1s**（与开机同） | **不存在该态**（已 ship mode → 必须 3s 走 POR） | Meta 只有一个"从 OFF 唤醒"接口，设计更统一 |
| 开机反馈 | 白灯脉动 | 黄常亮 + 振动 1 次 + STARTING 2s | 本项目交互反馈更多 |
| 误触防护 | 1s 阈值 + 必须在充电状态下 | 3s 阈值 + boot magic + SHPHLD 事件位双校验 | 本项目兜底更严（调试器热复位场景） |

### 4.2 关机

| 项 | Meta | 本项目 | 评注 |
|---|---|---|---|
| 主动关机长按 | **6s**（灯渐暗） | 8s（短振 2 次） | 时长接近；Meta 用 LED 渐暗作表现，本项目用振动 |
| 自动关机判据 | **IMU 无运动 1h** | 纯时间（3 min + 5 min） | Meta motion 判据避免静止误关 |
| 关机前用户提示 | LED 灯光逐渐熄灭 | 短振 2 次 | 不同感官，都可感知 |
| 关机静态电流 | 未公开（推测 µA 级） | ~0.5 µA（PMIC ship mode） |

**差异要点**：两者都**同时提供显式关机 + 自动关机**，差异主要在自动关机判据：Meta 用 IMU motion，本项目用连接状态 + 时间。本项目用户戴着但静止（开会）满 3+5 = 8 min 会被关机，可能误断。

### 4.3 配对

| 项 | Meta | 本项目 |
|---|---|---|
| 默认是否广告 | **否**（仅在配对窗口期广告） | **是**（WAITING 即广告，3 min 才退入 STANDBY） |
| 进入配对的明确动作 | 长按 **3s** | 无（开机即可被发现） |
| 配对中状态反馈 | LED **白灯闪烁** | LED 白心跳（1s 亮 / 1s 灭） |
| 配对成功反馈 | LED **白光转常亮** | LED 全灭（进 WORKING 省电）+ 振动 1 次 |
| 安全考量 | 防被陌生主机扫到 | 算法采集期为方便调试长期暴露 |

### 4.4 恢复出厂 / 故障兜底

| 项 | Meta | 本项目 |
|---|---|---|
| 长按时长 | **16s** | 15s（接近一致） |
| 反馈 | 橙光 + 白光脉动 | 白 / 黄 交替 + 短振 3 次 + 冷重启 |

### 4.5 LED 与振动

| 项 | Meta | 本项目 |
|---|---|---|
| LED 颜色 | 白 / 绿 / 橙 + 橙白脉动 | 白 / 黄 / 绿 / 红 / 白黄交替 |
| LED 表达维度 | **颜色 + 节奏（常亮 / 闪烁 / 脉动）** | 颜色 + 节奏（常亮 / 心跳 / 对称闪） |
| 表示状态数 | 配对中 / 已连 / 充满 / 未满 / 低电 / 出厂 = **6** | 开机 / 等连 / 工作 / 待机 / 充电多状态 / 低电多状态 / 出厂 = **≥10** |
| 亮度调节 | 未公开 | 软件 PWM 16 步 + R:G:B = 8:2:4 色温修正 |
| 电池可查询性 | **随时短按查看** | 仅在充电 / 低电时被动显示，不可主动查询 |
| 振动反馈 | 开机一次（可能也在其他事件，说明书未列） | 开机 / BLE 连接 / 关机 / 出厂 （1 / 1 / 2 / 3 次）|

**要点**：Meta 多了一个**主动查电量**的交互（短按 → 颜色反馈），本项目当前没有。

### 4.6 数据流与功耗

| 项 | Meta | 本项目 |
|---|---|---|
| 上行内容 | 离散事件（click / scroll / pinch …） | 原始 EMG（8ch × 24bit）+ IMU 原始 + 融合姿态 |
| BLE 带宽需求 | 极低（< 1 kbps 量级，事件驱动） | 高（EMG 长期 ~ 数百 kbps） |
| 主机依赖 | 已绑定的 Meta 眼镜 / Meta AI app | 任意支持本项目协议的上位机 / Android app |
| 续航 | 公开报告 ~ 18 h | 受 EMG 全量上行影响，目标态减负后再评估 |

## 5. 可借鉴的目标态调整建议

> 当前是**算法开发期**，调试信息密度高、长期广告、纯时间息屏都是有意为之。下面是**未来迁向 HID 部署态**时可参考 Meta 的点，仅作记录，不直接进 Backlog。

1. **Motion-based 自动关机**
   STANDBY 期间叠加 IMU 静止判据（例如：累计静止 > N 分钟 → 直接 ship mode），避免戴在手腕上但用户暂时不动也被强制关机
2. **配对态独立化**
   默认上电不进入广告；按特定按键长按进入"可被发现"窗口（30 ~ 60s），其余时间仅对已绑定主机定向重连。提升安全性 + 降低被动扫描功耗
3. **短按查电量**
   加入与 Meta 一致的交互：运行中短按 → 以 LED 颜色反馈当前电量（3 ~ 5s 后自恢复原模式）。本项目现状只在充电 / 低电时被动显示，用户不能主动查
4. **统一长按阶梯到 1 / 3 / 6 / 16s**
   Meta 的阶梯语义递进且骨感明确（越久越破坏性）；本项目 3 / 8 / 15s 中间跳跃大。概念上可改为 1s 短按查电 + 3s 进配对 + 6s 关机 + 15/16s 出厂
5. **HID 部署态简化 LED**
   保留白 / 绿 / 橙 三色 + 充电指示 + 配对状态，对齐 Meta；调试期预留的黄 / 红 / 绿多模式可裁剪
6. **关机动画从振动补充为 LED 渐暗**
   Meta 的 "灯光逐渐熄灭" 是低成本高质感反馈；本项目由于需在关机后进 ship mode，可考虑在 motor 震动后、ship mode 前加 200ms LED 渐暗过渡
7. **UI 上把"自动关机"行为暴露给用户**
   让用户预期"摘下来 N 分钟自动关"，而不是怀疑断电 / 故障
