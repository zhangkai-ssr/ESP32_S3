# nRF54L15 开关机与系统状态机逻辑（Meta Neural Band 对齐版）

> 平台：nRF54L15 + nPM1300 PMIC
> 按键时序参照 Meta Neural Band 实测：1s 开机 / 3s 配对 / 6s 关机 / 16s 恢复出厂
> 最后更新：2026-05-20

---

## 0. 按键时序总览（实测对齐表）

| 长按时间 | 行为 | LED | Motor | 触发条件 |
|---|---|---|---|---|
| **1s**  | 开机           | 白色慢脉动      | 短慢震             | 仅关机态：PMIC 硬件唤醒 (ship-to-active = 1008ms) → MCU POR |
| **3s**  | 进入配对模式   | 白色快脉动      | 长快震             | 关机态：1s 已开机后继续按到 3s；开机态：直接按到 3s |
| **6s**  | 关机           | 白灯缓慢淡出    | 短震 + 渐弱        | 仅开机态：到达 6s latch + 短震预览，**松开后 commit** `system_state_shutdown()` |
| **~10s**| (自然) 重新开机 | 白色慢脉动      | 短慢震             | 6s 关机后用户仍按住，PMIC ship-to-active (~1s) 后自动重上电 |
| **16s** | 恢复出厂       | 橙 + 白脉动     | 出厂震动 (4 次)    | 10s 重启后用户继续按 ~6s，新 boot 累计达到出厂阈值 |

> **关键设计**：将 PMIC `ship-to-active-time-ms` 从 `3008` 改为 `1008`，使硬件唤醒延迟与 Meta 1s 阈值一致。这是实现"1s 开机"唯一可行路径——nPM1300 SHPHLD 唤醒由 PMIC 内部计时器实现，软件无法绕过。
>
> **分级反馈机制**：每个阈值跨越时立即给出对应的 LED+马达反馈（latch 标志位防重复）。关机阈值不等待释放即执行；配对阈值需用户释放后才进入配对态（避免与 6s 关机冲突）。

## 2. 关机状态（OFF / Ship Mode）

### 2.1 PMIC (nPM1300) — Ship Mode

| 项目 | 状态 | 说明 |
|------|------|------|
| **工作模式**     | Ship mode (hibernate) | `mfd_npm1300_hibernate(pmic_dev, 0)` |
| **BUCK1**        | 关闭 | MCU 完全断电 |
| **LDO1 / LDO2**  | 关闭 | 外设全部断电 |
| **充电功能**     | 保持 | VBUS 接入时 PMIC 独立充电 |
| **SHPHLD 引脚**  | 监听中 | 唯一唤醒源 |
| **唤醒条件**     | SHPHLD 长按 ≥1008ms | DTS `ship-to-active-time-ms = <1008>` |
| **LED 驱动**     | 关闭 | 全灭 |
| **静态电流**     | ~0.5µA | nPM1300 ship 典型 |

### 2.2 MCU (nRF54L15) — 完全断电

| 项目 | 状态 |
|------|------|
| 供电  | VDD = 0V |
| RAM   | 丢失（`s_boot_magic` 不保持） |
| GPIO  | 高阻态 |
| RRAM  | 保持（固件代码不丢失） |

### 2.3 唤醒时序

```
   SHPHLD 按下                    SHPHLD 持续按住 ≥1008ms
       │                              │
       ▼                              ▼
   PMIC 启动 ship-to-active 计时   PMIC 退出 ship mode
                                  ├─ BUCK1 → MCU VDD 上电
       │  用户中途松开 (<1008ms)   ├─ LDO1/LDO2 → 外设上电
       │  → PMIC 取消，保持 ship    ├─ SHPHLD PRESS 事件写入寄存器
       │                          └─ MCU 执行 POR → main()
```

> 与原 3008ms 相比：1008ms 误触概率上升（口袋/搬运 ≥1s 即可开机）。可在 main.c 中 STARTING 阶段增加 200ms 软件二次确认作为可选缓解。

## 3. 开机流程 (`main.c`)

```
PMIC ship mode
    │ 用户长按 SHPHLD ≥1008ms
    ▼
┌─────────────────────────────────┐
│  MCU POR → main() 入口          │
└─────────────────────────────────┘
    │
    ├─ 1. 检查 s_boot_magic (.noinit RAM)
    │     ├─ == BOOT_MAGIC → 热复位 → enter_ship_mode → sys_reboot
    │     └─ != BOOT_MAGIC → 设置 magic（POR 路径）
    │
    ├─ 2. 读 PMIC SHPHLD 事件寄存器
    │     ├─ 无 PRESS 事件 → 非按键唤醒 → 重回 ship
    │     └─ 有 PRESS 事件 → 清除事件位，确认按键唤醒
    │
    ├─ 3. printk "[BOOT] SHPHLD wake confirmed, starting app"
    │
    ├─ 4. app_start() — 初始化所有子系统 (顺序关键)
    │     → MCUboot img confirm (首次启动自动确认, 防回滚)
    │     → board_io_init()       GPIO 初始化
    │     → pmic_npm1300_init()   PMIC 寄存器配置 + 充电 + LDO
    │     → button_init()         按键轮询线程 (依赖 PMIC)
    │     → motor_app_init()      马达驱动初始化 + 自检震动 (15% 150ms)
    │     → led_ctrl_init()       LED 控制初始化
    │     → flash_init()          外部 SPI Flash
    │     → output_init()         输出模块
    │     → led_app_init()        LED BLE 协议 (0x04) 注册
    │     → ads1298_init()        EMG 前端 SPI 初始化
    │     → ble_app_init()        BLE 协议栈 + 广播启动
    │     → emg_acq_start()       启动 EMG 采集
    │     → ~~imu_app_init()~~    IMU 初始化 (当前已注释掉)
    │     → system_state_init()   启动状态机 → SYS_STATE_STARTING
    │
    └─ 5. main 线程进入 while(1) { k_sleep(1s); }
```

## 4. 系统状态机 (`system_state.c`)

### 4.1 状态定义

| 状态 | 中文 | 枚举 | 说明 |
|------|------|------|------|
| `SYS_STATE_OFF`             | 关机     | 0 | 执行 `system_state_shutdown()` → PMIC ship mode |
| `SYS_STATE_STARTING`        | 开机中   | 1 | LED 全灭 + 马达短震 1 次 (30% 300ms)，2 秒后自动跳转 |
| `SYS_STATE_PAIRING`         | 配对中   | 2 | 清 bond 重启广播，60 秒超时回 WAITING_CONNECT |
| `SYS_STATE_WAITING_CONNECT` | 等待连接 | 3 | LED 全灭，BLE 广播中 |
| `SYS_STATE_WORKING`         | 工作中   | 4 | LED 全灭 (省电)，EMG 采集 + BLE 传输 |
| `SYS_STATE_STANDBY`         | 待机     | 5 | LED 全灭，低功耗 |

> **注意**：当前代码中所有状态入口动作的 LED 均设为 `LED_MODE_OFF`，`compute_power_led_mode()` 也固定返回 `LED_MODE_OFF`。LED 行为目前仅由 `button.c` 长按反馈和 `system_state_shutdown()` 渐灭序列驱动。

### 4.2 状态流转图

```
                    ┌──────────────────┐
                    │  PMIC ship mode  │
                    │     OFF (关机)   │
                    └────────┬─────────┘
                             │ SHPHLD ≥1s → POR
                             ▼
                    ┌──────────────────┐
                    │ STARTING (开机中)│ 白慢脉 + 短慢震
                    │   (2s 等待)      │
                    └────────┬─────────┘
                             │ 2s 超时
                             ▼
       ┌─────────── 长按 3s ─┤
       │                     │ ┌──────────────────┐
       ▼                     └▶│ WAITING_CONNECT  │ 白色心跳
┌──────────────┐               │  (等待连接)      │ BLE 广播
│  PAIRING     │  配对成功     └────┬────────┬───┘
│  (配对中)    │◀── 短按 ──────┐    │        │ 3min 超时
│ 白色快脉动   │   取消        │ BLE连接     │
│ 长快震       │               │    ▼        ▼
│ 清 bond +    │               │ ┌─────────┐ ┌─────────┐
│ 可配对广播   │───────────────┴▶│ WORKING │ │ STANDBY │ 黄色心跳
│ 60s 超时 ────┴──▶ WAITING     │ (工作中)│ │ (待机)  │
└──────────────┘                 │ EMG采集 │ └─┬───┬───┘
                                 └────┬────┘   │   │
                                      │ BLE断开│   │ 5min超时
                                      ▼        │   ▼
                                  WAITING ◀────┘  OFF
                                                 (ship mode)
   长按 6s (开机态) ─────────────▶ system_state_shutdown() → OFF
   (持续按住) ────── PMIC 自然 ship-to-active(~1s) ───────▶ POR → STARTING
   再按 ~6s (累计 ~16s) ──────▶ 恢复出厂 → cold reboot
```

### 4.3 状态线程

- **线程栈**：2048 字节
- **优先级**：6 (`SYS_THREAD_PRIO`)
- **轮询间隔**：200ms (`SYS_TICK_MS`)

> 按键轮询已独立到 `button.c` 线程 (优先级 7, 每 50ms 轮询 PMIC SHPHLD)。

每次循环执行：
1. `process_state()` — 处理原子标志 (`f_ble_connected` / `f_ble_disconnected` / `f_request_pairing` / `f_short_press`) 和状态超时
2. 状态发生变化时调用 `enter_xxx()` 入口动作

## 5. 各状态进入动作

### STARTING (开机中) — `enter_starting()`
```c
led_ctrl_force_off();                              // LED 全灭
led_ctrl_set_mode(LED_MODE_OFF, false);            // work_led 关
motor_app_feedback_power_on();                     // 30%, 300ms, 1次
// 2秒后自动 → WAITING_CONNECT
```

### PAIRING (配对中) — `enter_pairing()`
```c
ble_app_enter_pairing();                           // 清 bond + 重启可配对广播
led_ctrl_set_mode(LED_MODE_OFF, true);             // power_led 关
led_ctrl_set_mode(LED_MODE_OFF, false);            // work_led 关
motor_app_feedback_pairing();                      // 60%, 250ms, 1次
// 60s 超时 → WAITING_CONNECT; BLE 连接 → WORKING
```

> 注意：进入 PAIRING 的 LED 反馈 (`LED_MODE_PAIRING_WHITE_PULSE_FAST`) 在 `button.c` 3s 阈值处提前触发，`enter_pairing()` 本身不再重复设置 LED。

### WAITING_CONNECT (等待连接) — `enter_waiting_connect()`
```c
led_ctrl_set_mode(LED_MODE_OFF, true);             // power_led 关
led_ctrl_set_mode(LED_MODE_OFF, false);            // work_led 关
// BLE 已连接 → WORKING; 3min 超时 → STANDBY
```

> EMG/ADS1298 启停暂未在状态切换中控制 (后续 PR 接入)。

### WORKING (工作中) — `enter_working()`
```c
led_ctrl_set_mode(LED_MODE_OFF, true);             // power_led 关
led_ctrl_set_mode(LED_MODE_OFF, false);            // work_led 关, 省电
// 仅当从 WAITING_CONNECT / STANDBY 进入时:
motor_app_vibrate_pattern(40, 200, 0, 1);          // 连接反馈 40%, 200ms, 1次
```

### STANDBY (待机) — `enter_standby()`
```c
led_ctrl_set_mode(LED_MODE_OFF, true);             // power_led 关
led_ctrl_set_mode(LED_MODE_OFF, false);            // work_led 关
// 5min 超时 → OFF (关机)
```

### OFF (关机) — `system_state_shutdown()`
```c
// 1. LED + 马达反馈
led_ctrl_force_off();                              // LED 全灭
led_ctrl_set_mode(LED_MODE_SHUTDOWN_WHITE_FADE_OUT, false);  // 白色渐灭动画
motor_app_feedback_shutdown();                     // 60%, 150ms, 1次
k_msleep(2200);                                   // 等待渐灭动画完成

// 2. GPIO 低功耗配置（ADS1298引脚拉低防反向供电）
board_io_enter_ship_mode();

// 3. 进入 PMIC ship mode
pmic_npm1300_enter_ship_mode();

// 4. 调试器兜底：若 MCU 仍存活 → sys_reboot(COLD)
```

> 按键 6s 触发的关机走 `commit_shutdown_from_preview()` (button.c)，略有不同：跳过马达反馈 (6s 时已短震过)，仅播 `LED_MODE_SHUTDOWN_WHITE_FADE_OUT` 后 ship mode。

### 恢复出厂 (16s 累计)
```c
led_ctrl_set_mode(LED_MODE_FACTORY_ORANGE_WHITE_PULSE, true);   // 橙+白 250ms 交替
motor_app_vibrate_pattern(60, 200, 150, 4);                     // 出厂震动 4 次
// 清 bond + 清 NVS 配置 → cold reboot
```

## 6. 按键事件处理（重设计）

按键通过 PMIC SHPHLD 事件寄存器轮询（50ms 间隔）。新方案使用**累计按住时长 + 阈值 latch**，每跨越一个阈值即时给反馈。

### 6.1 阈值常量

```c
// button.c
#define SHORT_PRESS_MIN_MS          50     // 按键消抖下限
#define PAIRING_THRESHOLD_MS        3000   // 3s : 配对
#define SHUTDOWN_THRESHOLD_MS       6000   // 6s : 关机
#define FACTORY_RESET_MS            16000  // 16s: 恢复出厂

// system_state.c
#define STARTING_TIMEOUT_MS         2000   // 开机动画 → WAITING_CONNECT
#define PAIRING_TIMEOUT_MS          60000  // 配对超时 → WAITING_CONNECT
#define WAITING_TIMEOUT_MS          180000 // 3min → STANDBY
#define STANDBY_TIMEOUT_MS          300000 // 5min → OFF

// main.c
#define WAKE_FALLBACK_HOLD_MS       1000   // boot_guard 软件长按唤醒阈值
```

### 6.2 分级反馈伪代码

```c
static bool latch_pairing, latch_shutdown, latch_factory;

while (1) {
    btn = btn_poll();
    held = btn_get_hold_duration_ms();

    if (held >= FACTORY_RESET_MS && !latch_factory) {
        latch_factory = 1;
        play_factory_reset_feedback();   // 橙白脉 + 出厂震动
        do_factory_reset();              // 清 bond/NVS → cold reboot
    } else if (held >= SHUTDOWN_THRESHOLD_MS && !latch_shutdown
               && current_state != SYS_STATE_OFF) {
        latch_shutdown = 1;
        play_shutdown_preview();         // 短震预览 (不立即关机)
        pending_action = ACTION_SHUTDOWN;
    } else if (held >= PAIRING_THRESHOLD_MS && !latch_pairing) {
        latch_pairing = 1;
        play_pairing_feedback();         // 白快脉 + 长快震
        pending_action = ACTION_PAIRING;
    }

    if (btn_released()) {
        if (pending_action == ACTION_SHUTDOWN)
            commit_shutdown_from_preview();  // LED 渐灭 → ship mode
        else if (pending_action == ACTION_PAIRING)
            system_state_set(SYS_STATE_PAIRING);
        else if (held >= SHORT_PRESS_MIN_MS && held < PAIRING_THRESHOLD_MS)
            handle_short_press();        // STANDBY → WAITING；PAIRING → 取消
        reset_latches_and_action();
    }
}
```

### 6.3 SHPHLD 寄存器

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| SHPHLD_EVT_SET | MAIN(0x00):0x12 | bit0=PRESS, bit1=RELEASE |
| SHPHLD_EVT_CLR | MAIN(0x00):0x13 | 写 1 清除 |
| SHPHLD_STATUS  | MAIN(0x00):0x14 | bit0=按住实时电平 |

## 8. LED 指示汇总

> **当前状态**：`compute_power_led_mode()` 固定返回 `LED_MODE_OFF`，所有状态入口动作 LED 均为 OFF。LED 行为仅由按键反馈和关机序列驱动。

### 8.0 状态入口 LED (system_state.c)

| 状态 | LED 模式 | 实际效果 |
|------|----------|----------|
| STARTING (开机中)         | `LED_MODE_OFF` | 全灭 |
| PAIRING (配对中)          | `LED_MODE_OFF` | 全灭 (LED 由 button.c 3s 阈值提前设置) |
| WAITING_CONNECT (等待连接) | `LED_MODE_OFF` | 全灭 |
| WORKING (工作中)           | `LED_MODE_OFF` | 全灭 (省电) |
| STANDBY (待机)            | `LED_MODE_OFF` | 全灭 |

### 8.0b 按键 / 关机 / 出厂 LED (button.c)

| 场景 | LED 模式 | 颜色 + 节奏 |
|------|----------|-------------|
| 长按跨 3s | `LED_MODE_PAIRING_WHITE_PULSE_FAST` | 白色快脉动 (250ms ON / 250ms OFF) |
| 长按跨 6s | 沿用 3s 的白色快脉动 | 不切换, 仅短震反馈 |
| 6s 松开关机 | `LED_MODE_SHUTDOWN_WHITE_FADE_OUT` | 白色渐灭 (~2s) |
| 长按 ≥16s 出厂 | `LED_MODE_FACTORY_RESET` | 白/橙交替 gamma 脉动 |

### 8.0c 充电 / 电量 LED (预留，当前未启用)

| 场景 | LED 模式 | 颜色 + 节奏 | 状态 |
|------|----------|-------------|------|
| 充电中 (电量 ≥20%)        | `LED_MODE_POWER_YELLOW_ON`    | 黄色常亮 | 未启用 |
| 充电中 (电量 <20%)        | `LED_MODE_POWER_YELLOW_BLINK` | 黄色闪烁 | 未启用 |
| 充满                      | `LED_MODE_POWER_GREEN_ON`     | 绿色常亮 | 未启用 |
| 低电量 (≤20%, 未充电)     | `LED_MODE_POWER_RED_ON`       | 红色常亮 | 未启用 |
| 极低电量 (≤10%, 未充电)   | `LED_MODE_POWER_RED_BLINK`    | 红色闪烁 | 未启用 |

> **LED 优先级 (`led_ctrl.c`)**
> - `power_led` 层 (充电/电量) 优先级高于 `work_led` 层 (BLE/工作).
> - 当 `power_led == LED_MODE_OFF` 时, `work_led` 才被显示出来.
> - `compute_power_led_mode()` 当前**固定返回 OFF**，充电/电量 LED 待后续 PR 接入。

### 8.1 LED 节奏一览

| 节奏类型 | 周期 | ON | OFF | 适用模式 |
|---------|------|----|----|--------|
| 常亮      | —     | 持续  | —    | `*_ON` 系列 (POWER_YELLOW_ON / POWER_GREEN_ON / POWER_RED_ON / WORK_BLUE_ON ...) |
| 心跳闪烁  | 2 s   | 1 s  | 1 s  | `WORK_WHITE_BLINK` / `WORK_YELLOW_BLINK` / `WORK_BLUE_BLINK` |
| 对称闪烁  | 1 s   | 500 ms | 500 ms | `POWER_RED_BLINK` / `POWER_YELLOW_BLINK` / `FACTORY_RESET` |

> 历史: WAITING / STANDBY 早期为 1 s 三角波**呼吸灯**, 因 nPM1300 LED 仅支持 on/off,
> 软件 PWM 在 I2C 总线抖动下亮度阶梯肉眼可见, 改为更稳定干净的 "心跳式" 二值闪烁.

### 8.2 LED 硬件实现要点

- nPM1300 三路 LED 引脚 (LED0=G / LED1=B / LED2=R) 仅支持 **on/off**, 无硬件 PWM.
- 颜色通过通道组合: R+G=黄, R+G+B=白, 单通道=红/绿/蓝.
- 亮度通过 **软件 PWM** 实现: 独立 cooperative 线程 (优先级 -2) tick 1 ms,
  16 步分辨率 → 周期 16 ms (62.5 Hz), 高于人眼闪烁临界.
- 色温修正: 最大占空比 R:G:B = 8:2:4 (50% / 12.5% / 25%), 修正绿/蓝偏强.
- 心跳闪烁的 ON 阶段直接调 `led_driver_set_rgb()` 走色温上限, 不再走亮度缩放;
  避免低亮度时整数截断引起的颜色偏差.

## 10. BLE 事件与状态机交互

```
ble_app.c connected()    → system_state_on_ble_connected()    → f_ble_connected = 1
ble_app.c disconnected() → system_state_on_ble_disconnected() → f_ble_disconnected = 1
```

状态线程每 200ms 检查标志位并消费：
- **WAITING_CONNECT / STANDBY / PAIRING + BLE连接** → WORKING (工作中)
- **WORKING (工作中) + BLE断开** → WAITING_CONNECT (等待连接)
- **任意非 OFF/PAIRING 状态 + `f_request_pairing`** → PAIRING (配对中)
- **PAIRING / STANDBY + 短按** → WAITING_CONNECT (等待连接)

## 12. 与 nRF52832 项目对比

| 项目 | nRF52832 | nRF54L15 |

## 13. 与 Meta Neural Band 对比

| 阈值 | Meta Neural Band | 本设备实现 | 一致性 |
|------|------------------|-----------|--------|
| 1s 开机           | LED 白慢脉 + 短慢震 + 自动配对等待 | PMIC ship-to-active=1008ms 硬件触发；STARTING → WAITING_CONNECT | ✅ |
| 3s 配对           | LED 白快脉 + 长快震                 | latch + 释放后进入 PAIRING；清 bond + 可配对广播 | ✅ |
| 6s 关机           | LED 白渐灭 + 短震渐弱               | latch + 短震预览，松开后 commit shutdown           | ✅ |
| ~10s 重启         | LED 白慢脉 + 短慢震                 | PMIC 自然 ship-to-active 重上电（无需软件）       | ✅ |
| 16s 恢复出厂      | LED 橙白脉 + 出厂震动               | 重启后新 boot 累计 ~6s 触发                       | ✅ |

> **权衡**：`ship-to-active-time-ms` 由 3008ms 降至 1008ms 后，开机灵敏度提升但误触概率上升。如需进一步抑制误触，可在 STARTING 阶段加 200ms 软件二次确认（按键仍按住才继续，否则回 ship mode）。
